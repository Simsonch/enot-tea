# Реагирование на инциденты (MVP)

## Цель

Задать минимальный регламент реагирования на инциденты, влияющие на production в период MVP.

## Уровни критичности

- `SEV-1`: недоступен checkout/заказы, есть риск потери данных.
- `SEV-2`: сильная деградация, но есть обходной путь.
- `SEV-3`: некритичная ошибка с низким бизнес-влиянием.

## Шаги реагирования

1. Подтвердить масштаб и критичность.
2. Назначить ответственного за инцидент.
3. Ограничить влияние (rollback/feature toggle/hotfix).
4. Для SEV-1 обновлять статус каждые 30 минут.
5. Зафиксировать root cause и список action items.

## Минимальные источники алертов

- Проверки доступности health endpoint API.
- Логи ошибок приложения.
- Алерты на падение доли успешного создания заказов.
