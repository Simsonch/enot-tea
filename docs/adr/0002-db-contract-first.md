# ADR 0002: Сначала контракт БД

## Контекст

Согласованность заказов и остатков - ключевой бизнес-риск MVP.

## Решение

Определять схему и миграции до полной реализации функционала.
Для локальной разработки использовать PostgreSQL через Docker Compose.

Статус:
- PostgreSQL: `Approved`.
- Локальный запуск через Docker Compose: `Approved`.
- Инструмент миграций: `Prisma Migrate (Approved)`.
- ORM/DB-access слой: `Prisma Client (Approved)`.
- Интеграция в backend `apps/api` (NestJS): `Applied`.

## Последствия

- Снижает риск расхождений между API и БД.
- Требует раннего согласования доменных терминов и жизненного цикла статусов.
