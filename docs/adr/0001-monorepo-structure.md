# ADR 0001: Структура монорепозитория

## Контекст

Для MVP нужно параллельно развивать storefront, admin, API и общие модули.

## Решение

Целевая структура после старта реализации:
- `apps/*` для деплоимых приложений
- `packages/*` для общих компонентов и библиотек

Стартовый baseline для первого технического спринта:
- `apps/api` для backend API
- `apps/storefront` для публичной витрины
- `apps/admin` для админ-приложения
- `packages/shared` для общих контрактов и утилит

Статус: `Approved` (MVP baseline).

## Последствия

- Проще сопровождать общие контракты.
- Один CI-пайплайн может запускать проверки для всех приложений.
- Нужны строгие границы workspace, чтобы избежать излишней связанности.
